Cortex 操作手册

适用于 Cortex v0.1.0 · macOS + Windows

1. 安装

从官网下载区下载对应你系统的安装包：

  • macOS Apple Silicon（M1/M2/M3/M4）→ Cortex-0.1.0-arm64.dmg
  • macOS Intel → Cortex-0.1.0-x64.dmg
  • Windows 10/11 (x64) → Cortex-0.1.0-Setup.exe

macOS 首次启动放行

未购代码签名，Gatekeeper 会拦。任选一种放行方式：

  1. 双击 Cortex.app → 弹"无法验证开发者" → 点"完成"（不要"移到废纸篓"）
  2. 打开 系统设置 → 隐私与安全性，滚到底，看到 "Cortex 已被阻止" → 点 "仍要打开"

或者一行命令直接放行：

xattr -dr com.apple.quarantine /Applications/Cortex.app

Windows 首次启动放行

SmartScreen 拦截未签名应用。处理方式：

  1. 双击 Cortex-0.1.0-Setup.exe
  2. SmartScreen 提示"未识别的应用" → 点 "更多信息"
  3. 下方按钮 "仍要运行" → 安装
部分杀软可能误报 PyInstaller 打包二进制（UPX 压缩特征），请加白名单后再装。

首次启动需要等待

Python 后端 (cortex-core) 首次启动需解压依赖，会等 15–25 秒，显示"正在启动 Cortex 核心..."。之后每次启动 1–2 秒。

2. 激活

Cortex 启动后，如果未激活会先显示一个激活页（不激活无法启动实盘策略，但可以浏览界面 / 跑模拟盘）。

流程

  1. 激活页上方会显示 本机机器码，形如 ABCD-EFGH-IJKL-MNOP。点旁边"复制"按钮。
  2. 把机器码发给销售。
  3. 付款后会拿到一串 88 字符激活码，类似 xRq3pZm...==。
  4. 回到 Cortex 激活页，粘到 激活码 输入框（也可以点"从剪贴板粘贴"自动填入）。
  5. 点 激活 → 若机器码与激活码匹配，立即进入主界面。

常见错误

  • "激活码与本机不匹配" — 你拿到的码是为另一台机器签的。重新申请。
  • "激活码长度异常" — 复制时少了字符。重新复制完整的 88 字符。
  • "激活码格式无效" — 误粘了非激活码内容（比如 URL、错位的文本）。
一台电脑一个码，换机器需要重新申请。详见官网 FAQ。

3. 首次设置（Onboarding）

激活完成后，Cortex 会引导你做最少必需的配置。

  1. 选择模式：
    • 模拟盘 Paper — 用真行情 + 假资金（初始 10000 USDT）。任何新策略都建议先在这里跑一周。
    • 实盘 Live — 用真实交易所账户。直接进入交易所选择 + API 录入。
  2. 如果选实盘，下一步选交易所（Binance / OKX / Bybit），填 API key 并测试连通性。
  3. 完成。可以随时回到 设置 页改这些。

4. 交易所 API 配置

设置页 → 交易所 API Key。三家都一样的填法。

权限规则（最重要的安全条款）

  • ✅ 勾"现货 & 杠杆交易"、"合约交易"、"读取"
  • ❌ 绝对不要勾"提现" — Cortex 不需要提现权限，开了就是给自己埋雷
  • ✅ 建议绑定 IP 白名单到你电脑的公网 IP（家庭宽带 IP 会变，但绑了就比裸跑安全得多）

各家差异

Binance — 只需 API Key + Secret，两项。创建 API

OKX — 需要 API Key + Secret + Passphrase（创建 API 时自己设的独立密码），三项都要填。创建 API

Bybit — 只需 API Key + Secret。创建 API

模拟盘 / Testnet 勾选框

录入 API 时有一个"模拟盘 / Testnet"勾选框：

  • ✅ 勾上 — 用对应交易所的测试环境（OKX Demo / Binance Testnet / Bybit Testnet）的 API key
  • ❌ 不勾 — 用实盘账户的 API key
⚠️ 错勾会报 50101 APIKey does not match current environment。删了 key 重录，勾对再试。

测试连通性

填完后点 测试，Cortex 会调用交易所的 fetch_balance。成功即说明 API + 网络 + 权限三项都对。

5. AI 模型配置

设置页 → AI Key。可选配置，不配也能跑策略，只是没法用 AI 顾问。

  • OpenAI — 填 OpenAI 官方 sk-... key
  • Claude (Anthropic) — 填 anthropic sk-ant-... key

你的 AI key 直接发到对应官方 API，不经 Cortex 服务器中转（也根本没这个服务器）。

6. 通知（邮件 / Telegram）

策略下单、异常、平仓事件可推送到邮件或 Telegram。半夜出事不漏过。

邮件 (SMTP)

设置页 → 邮件通知。需要：

  • SMTP Host + Port（如 smtp.qq.com + 465）
  • 用户名 + 密码（注意 QQ / 163 等需要"授权码"而不是登录密码）
  • 发件人 + 收件人邮箱

填完点 发送测试邮件，收到说明配置 OK。

Telegram

设置页 → Telegram 通知。需要：

  1. 找 @BotFather 创建一个 bot，拿到 bot_token
  2. 给你的 bot 发任意消息
  3. 访问 https://api.telegram.org/bot<TOKEN>/getUpdates，复制 chat.id
  4. 把 bot_token + chat_id 填入设置页 → 点 发送测试消息

7. 模拟盘

设置页顶部有 "🎮 模拟盘钱包"，初始 10000 USDT。

  • 用真实交易所行情，但成交全部在 Cortex 内部模拟（不会真下单）
  • 支持点 重置钱包 把虚拟资产清零、USDT 回到 10000
  • 任何新策略 / 新参数都建议先在模拟盘跑一周，看胜率和回撤再上实盘

8. 策略：挑选与配置

从策略市场创建

侧边栏 策略市场 列出所有内置策略类型。点一个进入配置页：

  • 选交易所 + 交易对（如 BTC/USDT）
  • 选市场类型（spot 现货 / swap 永续）
  • 选模式（paper 模拟盘 / live 实盘）
  • 填策略专属参数（如网格上下限、步长、仓位百分比）

内置策略类型

  • 网格 Grid — 区间震荡行情，自动低买高卖
  • 趋势跟踪 Trend — 跟随均线突破方向
  • 均值回归 Mean Reversion — 偏离过远后等回归
  • 定投 DCA — 周期性买入，平摊成本
  • AI 自主交易 AI Trader — 由 AI 看图表 / 新闻给买卖信号

配置好后

点 创建，策略进入 策略列表（侧边栏"策略"项），状态为 created。还没跑。

9. 启动策略

启动前的 preflight 检查

点策略 → 详情页 → 启动。Cortex 会先跑一遍参数检查：

  • 阻断项 blocker（红） — 即使强制也不让跑。例如合约策略没设止损、网格上下限反了、仓位 100% 等
  • 警告 warning（黄） — 提醒但允许继续。例如步长太密、滑点设得很大等

warning 弹窗会让你点 "仍要启动" 二次确认。blocker 则必须改参数才能启动。

启动后

状态变成 running，左下角侧边栏会有跑动指示灯 + 计数器。所有下单、成交、错误都进入通知面板（右上角铃铛）。

10. 紧急控制 🛑

策略详情页右上角有个 🛑 紧急控制 按钮。出意外时用。

三级递进操作

  1. 撤销全部挂单 —— 撤所有未成交挂单，策略继续跑。
    适用：行情突变想清掉旧挂单换方向，但还想让策略接管后续
  2. 强制市价平仓 + 停止策略 —— 立刻按市价平掉持仓 + 停策略。
    需要在确认框里手动输入"强制平仓" 四个字才能执行（防误触）。
    适用：行情失控、需要立刻保命
  3. 仅停止策略 —— 停掉策略，但保留持仓和挂单。
    适用：想暂停让策略别再下新单，但仓位想自己手动管理

全局紧急停止

侧边栏右下角有个 "全部停止" 按钮，停掉所有运行中的策略。每个策略的 on_stop 会撤掉自己的挂单，所以安全。

11. 回测

侧边栏 回测。

  1. 选策略类型 + 交易对 + 时间区间
  2. 填策略参数
  3. 点 跑回测 → Cortex 拉历史 K 线，模拟整段历史的成交
  4. 结果包含：总收益、胜率、最大回撤、夏普比率、所有成交明细

参数寻优

选定一组想 sweep 的参数（如网格步长从 0.5% 到 2%，每 0.1% 一档），Cortex 会跑所有组合，给出收益曲面 + 最优组合推荐。

过拟合警告：在历史数据上找出来的"最优参数"不保证未来好用。建议把数据切成 in-sample / out-of-sample 两段，前段优化、后段验证。

12. AI 顾问

侧边栏 AI 顾问。前提：你已在设置里配了 OpenAI / Claude key。

  • 聊天界面，可问"现在 BTC 趋势怎么看"、"我这套网格参数合理吗"
  • 支持把当前策略状态、最近成交、当前持仓作为上下文传给 AI
  • 支持 streaming 输出（边生成边显示）

AI 给的是建议不是指令，最终决策由你。

13. 钱包视图

侧边栏 钱包。看你在每个交易所的余额、持仓、未实现盈亏。

  • 聚合所有已配置交易所的资产数据
  • 显示总资产估值（按 USDT 计）
  • 区分现货 / 合约持仓

14. 主控台

启动 Cortex 后默认看到的页面，聚合所有正在跑的策略：

  • 每个策略一卡片：名字、交易所、模式、当前盈亏、运行时长
  • 点卡片直接进策略详情页
  • 底部 分析摘要 区显示今日 / 本周 / 本月的累计盈亏 + 交易次数

15. 故障排查

启动卡在"正在启动 Cortex 核心..."

  • 正常现象（首次冷启动 15–25 秒）
  • 超过 30 秒还没好 → 看右上角是否有"Cortex 核心未启动"提示。重启 Cortex 一次。
  • 反复出现 → 检查代理 / VPN（cortex-core 只用 127.0.0.1）

OKX 报 50101 错误

APIKey does not match current environment — 模拟盘 key 错填到实盘，或反之。重新录入 key 时正确勾选"模拟盘 / Testnet"。

下单立刻被拒（Invalid Order）

  • 仓位太小 — 各交易所有最小下单金额（通常 5–10 USDT），仓位太小会被拒
  • 价格偏离 — 限价单价格离最新价过远，被风控拦
  • 权限不足 — 复查 API 是否勾了"交易"权限

SMTP 测试发不出去

  • QQ / 163 邮箱需要用"授权码"不是登录密码
  • 端口：SSL 用 465，STARTTLS 用 587
  • 开发者邮箱被锁定？登录邮箱 web 端解锁

Telegram 收不到消息

  • 你必须先给 bot 发过任意消息，否则 bot 不能主动联系你（Telegram 安全策略）
  • chat_id 写错了？重新去 getUpdates 接口验证

策略启动后没下单

  • 是否在实盘而 API 没钱？
  • 触发条件是否还没满足（如网格还没到价位、趋势策略还没确认信号）
  • 看通知面板有没有报错

清空所有本地数据重新开始

退出 Cortex，删除数据目录，重启即可（会回到 onboarding 流程，需要重新激活）：

# macOS
rm -rf ~/.cortex

# Windows
rmdir /s %USERPROFILE%\.cortex

  • 错误截图 / 通知面板内容
