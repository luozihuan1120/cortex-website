Cortex 操作手册

适用于 Cortex v1.1.0 · macOS + Windows

1. 安装

从官网下载区下载对应你系统的安装包：

  • macOS Apple Silicon（M1/M2/M3/M4）→ Cortex-1.1.0-arm64.dmg
  • macOS Intel → Cortex-1.1.0-x64.dmg
  • Windows 10/11 (x64) → Cortex-1.1.0-Setup.exe

macOS 首次启动放行

未购代码签名，Gatekeeper 会拦。任选一种放行方式：

  1. 双击 Cortex.app → 弹"无法验证开发者" → 点"完成"（不要"移到废纸篓"）
  2. 打开 系统设置 → 隐私与安全性，滚到底，看到 "Cortex 已被阻止" → 点 "仍要打开"

或者一行命令直接放行：

xattr -dr com.apple.quarantine /Applications/Cortex.app

Windows 首次启动放行

SmartScreen 拦截未签名应用。处理方式：

  1. 双击 Cortex-1.1.0-Setup.exe
  2. SmartScreen 提示"未识别的应用" → 点 "更多信息"
  3. 下方按钮 "仍要运行" → 安装
部分杀软可能误报 PyInstaller 打包二进制（UPX 压缩特征），请加白名单后再装。

首次启动需要等待

Python 后端 (cortex-core) 首次启动需解压依赖，会等 15–25 秒，显示"正在启动 Cortex 核心..."。之后每次启动 1–2 秒。

国内网络：交易所 API 走代理

Binance / OKX / Bybit 的 API 在国内被墙，必须开代理。Cortex v1.0.0 起加了网络自愈——侧边栏底部每 10 秒自检 Binance，状态变化用脉动指示器实时显示：

  • 🟢 网络正常 + 延迟 — 可以正常下单/拉行情
  • 🔴 网络异常 · 点击查看 — 点击会显示当前检测到的代理 + 设置指引

使用方法：

  1. 装 ClashX / Surge / V2Box 任一
  2. 导入订阅 → 启动
  3. 菜单栏图标 → 勾选 「设置为系统代理」（关键）

不需要重启 Cortex：Cortex 检测到代理后会自动重连，10 秒内侧边栏变绿，运行中的策略自动续跑。

2. 激活

Cortex 启动后，如果未激活会先显示一个激活页（不激活无法启动实盘策略，但可以浏览界面 / 跑模拟盘）。

流程

  1. 激活页上方会显示 本机机器码，形如 ABCD-EFGH-IJKL-MNOP。点旁边"复制"按钮。
  2. 把机器码发给销售。
  3. 付款后会拿到一串 88 字符激活码，类似 xRq3pZm...==。
  4. 回到 Cortex 激活页，粘到 激活码 输入框（也可以点"从剪贴板粘贴"自动填入）。
  5. 点 激活 → 若机器码与激活码匹配，立即进入主界面。

常见错误

  • "激活码与本机不匹配" — 你拿到的码是为另一台机器签的。重新申请。
  • "激活码长度异常" — 复制时少了字符。重新复制完整的 88 字符。
  • "激活码格式无效" — 误粘了非激活码内容（比如 URL、错位的文本）。
一台电脑一个码，换机器需要重新申请。详见官网 FAQ。

3. 首次设置（Onboarding）

激活完成后，Cortex 会引导你做最少必需的配置。

  1. 选择模式：
    • 模拟盘 Paper — 用真行情 + 假资金（初始 10000 USDT）。任何新策略都建议先在这里跑一周。
    • 实盘 Live — 用真实交易所账户。直接进入交易所选择 + API 录入。
  2. 如果选实盘，下一步选交易所（Binance / OKX / Bybit），填 API key 并测试连通性。
  3. 完成。可以随时回到 设置 页改这些。

4. 交易所 API 配置

设置页 → 💰 实盘交易所 API Key。三家都一样的填法。

权限规则（最重要的安全条款）

  • ✅ 勾"现货 & 杠杆交易"、"合约交易"、"读取"
  • ❌ 绝对不要勾"提现" — Cortex 不需要提现权限，开了就是给自己埋雷
  • ✅ 建议绑定 IP 白名单到你电脑的公网 IP（家庭宽带 IP 会变，但绑了就比裸跑安全得多）

各家差异

Binance — 只需 API Key + Secret，两项。创建 API

OKX — 需要 API Key + Secret + Passphrase（创建 API 时自己设的独立密码），三项都要填。创建 API

Bybit — 只需 API Key + Secret。创建 API

模拟盘 / Testnet 勾选框

录入 API 时有一个"模拟盘 / Testnet"勾选框：

  • ✅ 勾上 — 用对应交易所的测试环境（OKX Demo / Binance Testnet / Bybit Testnet）的 API key
  • ❌ 不勾 — 用实盘账户的 API key
⚠️ 错勾会报 50101 APIKey does not match current environment。删了 key 重录，勾对再试。

测试连通性

填完后点 测试，Cortex 会调用交易所的 fetch_balance。成功即说明 API + 网络 + 权限三项都对。

5. AI 模型配置

设置页 → 🤖 AI 平台 API Key。可选配置，不配也能跑策略，只是用不了 AI 分析。

支持的 5 家 AI 平台（2026-06 最新）

  • OpenAI — sk-... key，主推 gpt-5.4-mini（默认）/ gpt-5.5 / gpt-5.5-pro
  • Claude (Anthropic) — sk-ant-... key，默认 claude-sonnet-4-6
  • DeepSeek — 国内/海外都能用，默认 deepseek-v4-flash（便宜）；旗舰 deepseek-v4-pro
  • 通义千问 (DashScope) — 国内首选，默认 qwen3.5-plus
  • 豆包 (Volcano Engine Ark) — 字节出品，默认 doubao-seed-2-0-lite-260506（直接传模型名即可）

自定义模型 ID

在 AI 分析 页和 AI 自主交易 策略的配置中，「模型」字段是 select + 自定义组合：

  • 下拉默认列出官方支持的全部模型
  • 最后一项 「+ 自定义 ID...」切到 text input，可手填任意官方支持的模型 ID（包括预览版、试验版等没在下拉里出现的）
  • 豆包用户也可填 ep-xxxxxx 形式的 Endpoint ID（高阶用法，可在 Ark 控制台配限流/计费）

你的 AI key 直接发到对应官方 API，不经 Cortex 服务器中转（也根本没这个服务器）。

6. 通知（邮件 / Telegram）

策略下单、异常、平仓事件可推送到邮件或 Telegram。半夜出事不漏过。

邮件 (SMTP)

设置页 → 📬 邮件通知。需要：

  • SMTP Host + Port（如 smtp.qq.com + 465）
  • 用户名 + 密码（注意 QQ / 163 等需要"授权码"而不是登录密码）
  • 发件人 + 收件人邮箱

填完点 发送测试邮件，收到说明配置 OK。

Telegram

设置页 → ✈️ Telegram 通知。需要：

  1. 找 @BotFather 创建一个 bot，拿到 bot_token
  2. 给你的 bot 发任意消息
  3. 访问 https://api.telegram.org/bot<TOKEN>/getUpdates，复制 chat.id
  4. 把 bot_token + chat_id 填入设置页 → 点 发送测试消息

7. 模拟盘

设置页顶部 🎮 模拟盘钱包，初始 10000 USDT。

  • 用真实交易所行情，但成交全部在 Cortex 内部模拟（不会真下单）
  • 支持点 重置钱包 把虚拟资产清零、USDT 回到 10000
  • 任何新策略 / 新参数都建议先在模拟盘跑一周，看胜率和回撤再上实盘

8. 外观主题

设置页 → 外观主题。支持深色 / 浅色 / 跟随系统三档。改完立即生效，无需重启。

9. 数据导出

设置页 → 📦 数据导出。把你的策略 / 订单 / 成交记录全量导出成 JSON 文件，便于：

  • 跨机器迁移（换电脑后导回）
  • 本地存档 / 报税
  • 外部分析（导入 Excel / Python pandas 自己分析）

导出文件包含**策略配置、订单流水、成交记录、AI 决策历史**，不包含你的 API key / AI key（那些永远不离开你的电脑）。

10. 策略：挑选与配置

从策略市场创建

侧边栏 策略市场 列出所有内置策略类型。点一个进入配置页：

  • 选交易所 + 交易对（如 BTC/USDT）
  • 选市场类型（spot 现货 / swap 永续）
  • 选模式（paper 模拟盘 / live 实盘）
  • 填策略专属参数（如网格上下限、步长、仓位百分比）

内置策略类型

策略市场分两组：稳定策略（低/中风险，可任意尝试）和高级实验（高风险，启动需三重确认）。

稳定策略

  • ⊞ 现货网格 Grid · 低风险 — 震荡市印钞机。在指定价格区间内分格挂单，低买高卖，靠波动赚差价。最适合横盘震荡行情。
  • ◷ 定投 DCA · 低风险 — 懒人最爱。按固定时间间隔买入固定金额，不择时、不焦虑。长期主义的最佳选择。
  • 〰 双均线 MA Cross · 中风险 — 趋势跟随经典。快速均线上穿慢速均线 = 金叉买入；下穿 = 死叉卖出。适合趋势性强的行情。

高级实验（请先在模拟盘跑透）

  • ⚡ 马丁加仓 Martingale · 高风险 — 逢跌加仓、均价拉低，反弹即获利。支持现货 / 合约（仅做多）。合约带杠杆放大盈亏，遇到趋势大跌可能爆仓。
  • ✦ AI 自主交易 AI Trader · 高风险（Beta） — 让大模型按你设定的边界自动决策买卖。AI 可能误判、可能违反预期、可能短时密集下单。强烈建议先用模拟盘跑 1 周。盈亏自负。

K 线周期

从 v1.0.0 起，所有需要选周期的策略（双均线、AI 自主交易）都支持 11 档周期：

1m  3m  5m  15m  30m  1h  2h  4h  6h  12h  1d

回测页和 AI 分析页也是同一套周期。

配置好后 — 创建时就会自动校验

点 创建，Cortex 会先跑一遍参数 preflight：

  • blocker（红） — 严重错误，立即拦截，不允许保存。例如网格上限 ≤ 下限、双均线 fast ≥ slow、合约 AI 没设止损等
  • warning（黄） — 不合理但还能跑，弹确认对话框让你二次确认。例如网格区间太窄、马丁倍投偏激进、DCA 间隔太短等

校验通过后策略进入侧边栏 我的策略 页，状态为 created。编辑参数页保存时同样会校验。

11. 启动策略

启动按钮

策略详情页右上角 ▶ 启动。状态会立刻切到 🟢 运行中（脉动绿色徽章），按钮变 ■ 停止。

状态徽章一览

  • 🟢 运行中（脉动） — 策略正在调度
  • 🔴 错误 — on_start 失败或 tick 异常，详见通知面板
  • 🟡 已暂停 — 系统检测到外部干预 / 连败冷却等
  • ⚪ 已停止 / 未启动 — 不在调度

启动后

侧边栏 🛑 紧急停止 (N) 按钮会显示运行中策略数。所有下单、成交、错误都进入通知面板（右上角铃铛）。

12. 紧急控制 🛑

策略详情页右上角有个 🛑 紧急控制 按钮（鼠标悬停会显示「紧急控制：撤销挂单 / 强制平仓」）。出意外时用。

三级递进操作

  1. 撤销所有挂单 —— 撤所有未成交挂单，策略继续跑。
    适用：行情突变想清掉旧挂单换方向，但还想让策略接管后续
  2. 强制市价平仓 + 停止策略 —— 立刻按市价平掉持仓 + 停策略。
    需要在确认框里手动输入"强制平仓" 四个字才能执行（防误触）。
    适用：行情失控、需要立刻保命
  3. 仅停止策略（保留持仓） —— 停掉策略，但保留持仓和挂单。
    适用：想暂停让策略别再下新单，但仓位想自己手动管理

全局紧急停止

侧边栏底部有个 🛑 紧急停止 (N) 按钮（N 是当前运行中的策略数），点击后会停掉所有运行中的策略并撤销挂单。需要二次确认。

13. 回测

侧边栏 回测。

  1. 选策略类型 + 交易对 + 时间区间
  2. 填策略参数
  3. 点 跑回测 → Cortex 拉历史 K 线，模拟整段历史的成交
  4. 结果包含：总收益、胜率、最大回撤、夏普比率、所有成交明细

参数寻优

选定一组想 sweep 的参数（如网格步长从 0.5% 到 2%，每 0.1% 一档），Cortex 会跑所有组合，给出收益曲面 + 最优组合推荐。

过拟合警告：在历史数据上找出来的"最优参数"不保证未来好用。建议把数据切成 in-sample / out-of-sample 两段，前段优化、后段验证。

14. AI 分析

侧边栏 AI 分析，页面标题 AI 智能分析。前提：你已在设置里配了任一家 AI 平台的 API key（OpenAI / Claude / DeepSeek / Qwen / 豆包）。

输入

  • 选 AI 厂商 + 模型（下拉 + 可自定义 ID）
  • 选交易所 + 交易对 + K 线周期（11 档可选，最细 1m）
  • 补充说明：自由文本，比如"我已持有现货仓位想加仓"

Cortex 会自动注入这些上下文

  • 最近 30 根 K 线（OHLCV）
  • 当前盘口 ticker
  • 你在该交易所 + 该币种的真实持仓（现货余额 + 合约持仓 + 均价 + 未实现盈亏） — 前提是该交易所配了 API key

AI 会基于真实仓位给"加仓 / 减仓 / 持有"的具体建议，而不是只讲行情。

输出

  • Streaming 输出（边生成边显示）
  • 覆盖：价格行为 · 量价关系 · 短期方向偏向 · 主要风险 · 仓位建议

AI 给的是建议不是指令，最终决策由你。

15. 钱包

侧边栏 钱包，页面标题 💰 钱包总览。看你在每个交易所的余额、持仓、未实现盈亏。

  • 聚合所有已配置交易所的资产数据
  • 显示总资产估值（按 USDT 计）
  • 区分现货 / 合约持仓

总估值覆盖范围（v1.0.0 起）

对每个有余额的非稳定币，Cortex 都会去交易所拉 ASSET/USDT ticker 来估值，不再限于 30 个主流币白名单。SHIB / PEPE / FLOKI 等长尾币也能正确算进总值。

拉不到价的（已下架币、新发币、稳定币变种等）会用 ℹ️ 灰色提示列出，仅作信息提示不影响总值显示。

错误分级

  • ⚠️ 黄色"查询失败" — 真错误（如 fetch_positions 权限不足、网络问题等），点击展开可见原因
  • ℹ️ 灰色"提示" — 信息级（如未计入总估值的小币），不影响交易

16. 总览

侧边栏 总览，启动 Cortex 后默认看到的页面，聚合所有正在跑的策略：

  • 每个策略一卡片：名字、交易所、模式、当前盈亏、运行时长
  • 点卡片直接进策略详情页
  • 底部 分析摘要 区显示今日 / 本周 / 本月的累计盈亏 + 交易次数

17. 故障排查

侧边栏底部"🔴 网络异常"

  • 国内访问交易所 API 被墙，必须开代理（详见第 1 章 国内网络一节）
  • 点击红色指示器会显示 Cortex 当前检测到的代理
  • 开了 ClashX 但还是红色？检查菜单栏图标是否勾了「设置为系统代理」
  • 开了代理 10 秒内会自动恢复，无需重启 Cortex；侧边栏变绿后策略自动续跑

启动卡在"正在启动 Cortex 核心..."

  • 正常现象（首次冷启动 15–25 秒）
  • 超过 30 秒还没好 → 看右上角是否有"Cortex 核心未启动"提示。重启 Cortex 一次。
  • 反复出现 → 检查代理 / VPN（cortex-core 只用 127.0.0.1）

策略启动了状态却变 🔴 错误

v1.0.0 起，纯网络问题不会让策略进 error（保持 🟢 运行中，下个 tick 自动重试）。如果策略变红了，原因大概率是：

  • 余额不足 — 实盘账户没钱，或模拟盘 USDT 用完了
  • API key 无效 / 权限不足 — 重新录入 + 测试连接
  • 配置问题 — 比如合约策略没设止损、网格上下限错位（preflight 通常已经拦了）
  • 交易所参数错 — 比如最小下单金额低于限制

具体原因可在右上角铃铛通知面板看到（v1.0.0 起策略进 error 会推送 critical 通知，含错误原因）。

OKX 报 50101 错误

APIKey does not match current environment — 模拟盘 key 错填到实盘，或反之。重新录入 key 时正确勾选"模拟盘 / Testnet"。

下单立刻被拒（Invalid Order）

  • 仓位太小 — 各交易所有最小下单金额（通常 5–10 USDT），仓位太小会被拒
  • 价格偏离 — 限价单价格离最新价过远，被风控拦
  • 权限不足 — 复查 API 是否勾了"交易"权限

SMTP 测试发不出去

  • QQ / 163 邮箱需要用"授权码"不是登录密码
  • 端口：SSL 用 465，STARTTLS 用 587
  • 开发者邮箱被锁定？登录邮箱 web 端解锁

Telegram 收不到消息

  • 你必须先给 bot 发过任意消息，否则 bot 不能主动联系你（Telegram 安全策略）
  • chat_id 写错了？重新去 getUpdates 接口验证

策略启动后没下单

  • 是否在实盘而 API 没钱？
  • 触发条件是否还没满足（如网格还没到价位、趋势策略还没确认信号）
  • 看通知面板有没有报错

清空所有本地数据重新开始

退出 Cortex，删除数据目录，重启即可（会回到 onboarding 流程，需要重新激活）：

# macOS
rm -rf ~/.cortex

# Windows
rmdir /s %USERPROFILE%\.cortex
